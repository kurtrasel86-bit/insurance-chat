ЗАДАНИЕ: Наведение порядка в базе знаний

1. РЕЗЕРВНАЯ КОПИЯ
   - Создать backup базы данных перед началом работ
   - Сохранить в файл с датой и временем

2. АНАЛИЗ ДОКУМЕНТОВ
   - Получить полный список всех документов
   - Проанализировать каждый документ по критериям ниже

3. КРИТЕРИИ ОЧИСТКИ

   А) НЕАКТУАЛЬНЫЕ (пометить как obsolete) ТОЛЬКО при явных признаках:
   - Документы с текстом "УСТАРЕЛО", "НЕ ДЕЙСТВИТЕЛЬНО", "ЗАМЕНЕНО"
   - Документы с указанием конкретной даты окончания действия
   - Документы с пометкой "ВЫПУЩЕН НОВЫЙ ДОКУМЕНТ"
   - Документы с явно противоречивой информацией
   - Временные акции с истекшим сроком (если срок указан)
   
   НЕ помечать как obsolete без явных признаков:
   - Документы только по возрасту
   - Правила страхования без пометок об устаревании
   - Общие условия и описания продуктов
   - Документы без четких указаний на неактуальность

   Б) НЕПОДХОДЯЩИЕ ДЛЯ СЕРВИСА (удалить):
   - Документы не по страхованию
   - Техническая документация для внутреннего использования
   - Дубликаты с низким качеством
   - Документы менее 100 символов

   В) ИСПРАВЛЕНИЕ НЕСОСТЫКОВОК:
   - Неправильная компания: проанализировать текст и исправить companyCode
   - Неправильный продукт: проанализировать содержание и исправить productCode
   - Неточные заголовки: исправить title на основе содержания

4. ПРОВЕРКА КАЧЕСТВА
   - Убедиться, что все документы имеют корректные метаданные
   - Проверить, что нет пустых или поврежденных документов
   - Валидировать коды компаний и продуктов

5. ОТЧЕТ
   - Количество обработанных документов
   - Количество помеченных как obsolete
   - Количество удаленных документов
   - Количество исправленных несостыковок
